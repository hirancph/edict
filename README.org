* Overview

This is my fully declarative GNU Guix system and home configuration, written
entirely in GNU Guile.

* Directory Structure

| Directory      | Purpose                                                |
|----------------+--------------------------------------------------------|
| =channels/=      | Channel signing keys and per-channel configuration     |
| =files/=         | Static assets that cannot be expressed in Guile        |
| =manifests/=     | Ad-hoc package manifests for =guix shell -m=             |
| =modules/edict/= | Guile module tree — the heart of the configuration     |
| =patches/=       | Source patches for upstream or custom packages         |
| =scripts/=       | Helper shell scripts for bootstrapping and maintenance |

** Guile Modules (=modules/edict/=)

| Module Path    | Purpose                                          |
|----------------+--------------------------------------------------|
| =config.scm=     | Global constants (user, timezone, locale, paths) |
| =utils.scm=      | Shared utility procedures                        |
| =features/=      | Composable feature units (packages + services)   |
| =home/=          | Home environment definitions                     |
| =home/services/= | Custom Guix Home service types                   |
| =packages/=      | Custom or patched package definitions            |
| =services/=      | Custom system-level service types                |
| =systems/=       | Per-host operating-system configurations         |

* Quick Start

#+begin_src sh
# Pull channels
make pull

# Reconfigure the system
sudo make system

# Reconfigure the home environment
make home
#+end_src

* Adding a New Feature

1. Create a new file in =modules/edict/features/=
2. Define your feature as a Guile module: =(define-module (edict features your-feature) ...)=
3. Bundle packages, services, and configuration within it
4. Import and compose it in your system or home configuration

* Adding a Custom Package

1. Create a new file in =modules/edict/packages/=
2. Define the module: =(define-module (edict packages your-package) ...)=
3. Use it anywhere via =(use-modules (edict packages your-package))=

* Adding a New Host

1. Create a new file in =modules/edict/systems/= (e.g., =laptop.scm=)
2. Define the =operating-system= for that host
3. Add a Makefile target or reconfigure command for it

* Design Principles

- *Fully Guile*: Everything that can be expressed in Guile, is.
- *Fully Declarative*: The system is entirely reproducible from this repo.
- *Composable*: Features are independent, reusable units.
- *Extensible*: Adding hosts, features, packages, or services follows a
  consistent pattern.
