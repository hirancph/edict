#+title: edict/

* Purpose

This is your main Guile namespace: `(edict ...)`.

Every `.scm` file inside this directory and its subdirectories is a Guile module
that belongs to the `edict` project.

* For Beginners: What is a Module?

If you open any file in this folder, the very first line is always:
#+begin_src scheme
(define-module (edict some-folder some-file))
#+end_src
This simply tells Guile (the engine): "Hey, this file is a package of code named exactly where it lives."

Right below it, you will see things like:
#+begin_src scheme
  #:use-module (gnu)
  #:export (my-thing)
#+end_src
- `#:use-module` means "I need to borrow tools from another file to do my work".
- `#:export` means "I am making this specific tool available so *other* files can borrow it from me".

* Structure

#+begin_example
modules/edict/
├── README.org       ;; This file
├── config.scm       ;; Global configuration variables `(edict config)`
├── utils.scm        ;; Shared helper procedures `(edict utils)`
├── features.scm     ;; ★ The feature engine `(edict features)`
├── build.scm        ;; OS & home-environment builders `(edict build)`
├── features/        ;; Composable feature modules `(edict features ...)`
├── hosts/           ;; Per-host feature selections `(edict hosts ...)`
├── home/            ;; Home configurations `(edict home ...)`
├── packages/        ;; Custom package definitions `(edict packages ...)`
├── services/        ;; Custom system services `(edict services ...)`
└── systems/         ;; Host OS configurations `(edict systems ...)`
#+end_example

* Core Files

** `config.scm`
Defines global constants like your username, timezone, locale, and paths.
By keeping these in one file, any other module can simply import `(edict config)`
to get your username instead of hard-coding it.

** `utils.scm`
Provides general-purpose helper functions used across the codebase, such as
the `pkgs` macro to turn strings into package objects, avoiding repetitive typing.
