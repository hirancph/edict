#+title: features/

* Purpose

Composable feature units built on the =edict-feature= record from =(edict features)=.

Each feature bundles kernel arguments, loadable modules, packages,
services, system groups, and user supplementary groups into a single
composable unit.  Multiple features are merged with =compose-features=;
duplicates in user groups are removed automatically.

* Design Pattern

Each feature module should:

1. Define a Guile module: =(define-module (edict features foo) ...)=
2. Import =(edict features)= for =edict-feature=, =contribute=, =ensure-pred=
3. Export a constructor function: =(define* (foo-feature #:key ...) ...)=
4. Validate all keyword arguments with =ensure-pred=
5. Set an appropriate =#:scope= (='system=, ='home=, or ='both=)

* Constructor Pattern

#+begin_src scheme
(define* (my-feature #:key
                     (option-a "default")
                     (option-b #t))
  "Docstring explaining what this feature does."

  ;; ── Validation ──
  (ensure-pred string? option-a)
  (ensure-pred boolean? option-b)

  (edict-feature
   #:name 'my-feature
   #:provides '(my-feature)      ;; Symbols this feature makes available
   #:requires '(base)            ;; Symbols that must be provided first
   #:scope 'both                 ;; 'system, 'home, or 'both
   #:values `((my-key . ,option-a))
   #:extensions
   (list
    (contribute system-packages-target ...)
    (contribute system-services-target ...)
    (contribute home-packages-target ...)
    (contribute home-services-target ...))))
#+end_src

* Available Targets

| Target                     | Wired into                  |
|----------------------------+-----------------------------|
| =system-packages-target=    | =operating-system= packages   |
| =system-services-target=    | =operating-system= services   |
| =home-packages-target=      | =home-environment= packages   |
| =home-services-target=      | =home-environment= services   |
| =kernel-arguments-target=   | Kernel command line          |
| =kernel-modules-target=     | Loadable kernel modules      |
| =groups-target=              | System group records         |
| =user-groups-target=         | User supplementary groups    |
| =user-accounts-target=      | Additional user accounts     |

* Value Publishing

Use =make-feature-values= for ergonomic value publishing:

#+begin_src scheme
;; Instead of:
#:values `((kernel . ,kernel) (firmware . ,firmware))

;; Use:
#:values (make-feature-values kernel firmware)
#+end_src

* Feature Scoping

Features can declare their scope:
- ='system= — only included in =edict-operating-system= builds
- ='home= — only included in =edict-home-environment= builds
- ='both= (default) — included in both

System-only features (like =nonguix=, =nvidia=, =ssh=) should use =#:scope 'system=.

* Features vs Packages vs Services

- *Features* (=features/=): High-level compositions — "I want desktop support"
- *Packages* (=packages/=): Custom package definitions not in upstream Guix
- *Services* (=services/=): Custom service type definitions not in upstream Guix

Features /use/ packages and services.  They are the glue layer.
