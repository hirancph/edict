#+title: systems/

* Purpose

Per-host =operating-system= configurations.

Each file defines a complete system configuration for a specific machine.
Currently there is only one host (vessel), but this structure supports
adding laptops, servers, or VMs in the future.

* What Goes Here

- =operating-system= definitions: kernel, bootloader, file systems, hostname,
  users, system packages, and system services
- One file per host machine

* For Beginners: What is `edict-operating-system`?

In `systems/vessel.scm`, you will see:
#+begin_src scheme
(edict-operating-system %vessel
  (host-name "vessel")
  (file-systems ...))
#+end_src

This looks like a function call, but it's actually a **Macro**. 
A macro is code that writes code for you. 

When you use `edict-operating-system`, it takes your composed features (`%vessel`), extracts all the packages, services, and kernel arguments that your features asked for, and automatically writes the massive `operating-system` block that Guix expects. 

You only need to define the hardware-specific things here (like your hostname, bootloader, and file systems) because those physical things cannot be abstracted away into reusable features.

* Example

#+begin_src scheme
;; modules/edict/systems/vessel.scm
(define-module (edict systems vessel)
  #:use-module (gnu)
  #:use-module (gnu system)
  #:use-module (edict config)
  #:use-module (edict features desktop)
  #:use-module (edict features networking))

(operating-system
  (host-name "vessel")
  (timezone %timezone)
  (locale %locale)
  (bootloader (bootloader-configuration
               (bootloader grub-efi-bootloader)
               (targets '("/boot/efi"))))
  (file-systems (cons (file-system
                        (device (uuid "..."))
                        (mount-point "/")
                        (type "ext4"))
                      %base-file-systems))
  (users (cons (user-account
                (name %user-name)
                (group "users")
                (supplementary-groups '("wheel" "audio" "video" "netdev")))
               %base-user-accounts))
  (packages (append %desktop-packages %base-packages))
  (services (append %desktop-services %base-services)))
#+end_src

* Naming Convention

#+begin_example
systems/
├── vessel.scm           ;; Main desktop workstation
├── laptop.scm           ;; (future) Laptop
├── server.scm           ;; (future) Home server
└── install.scm          ;; (future) Installation ISO
#+end_example

* Reconfiguring

#+begin_src sh
sudo guix system reconfigure -L modules/ modules/edict/systems/vessel.scm
# Or use the Makefile:
sudo make system HOST=vessel
#+end_src
