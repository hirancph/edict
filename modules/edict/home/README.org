#+title: home/

* Purpose

Home environment definitions and custom Guix Home services.

This is where user-level (non-root) configurations live, as opposed to
system-level configurations in =systems/=.

* For Beginners: System vs Home

Guix is unique because it completely separates your Operating System from your User Profile.

**System (=sudo make system=)**: 
Controls hardware, kernel, drivers, networking, and system-wide services. If you break this, your computer might not boot.

**Home (=make home=)**: 
Controls your personal user account. This includes your dotfiles, your window manager theme, your custom bash prompt, and applications that only *you* use (like Discord or Emacs). You do not need =sudo= for this. Since it only affects your user account, it is impossible to break the whole computer.

* Structure

#+begin_example
home/
├── README.org          ;; This file
├── vessel.scm          ;; Home environment for vessel
└── services/           ;; Custom home service type definitions
    └── README.org
#+end_example

* What Goes Here

- =home-environment= definitions that compose features into a complete
  user environment for a specific host
- Host-specific home configurations (one =.scm= per host)

* Example

#+begin_src scheme
;; modules/edict/home/vessel.scm
(define-module (edict home vessel)
  #:use-module (gnu home)
  #:use-module (edict config)
  #:use-module (edict features shells)
  #:use-module (edict features desktop))

(home-environment
  (packages (append %desktop-packages %shell-packages))
  (services (append %desktop-home-services %shell-services)))
#+end_src

* Home vs System

| Concern | Where |
|---------+-------|
| User packages & dotfiles | =home/= |
| Desktop environment | =home/= |
| Kernel, bootloader, filesystems | =systems/= |
| System daemons | =systems/= or =services/= |
